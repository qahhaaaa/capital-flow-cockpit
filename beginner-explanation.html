流动性信号面板小白理解版

先看钱在哪，再判断行情热不热。

这套面板不是“预测明天涨跌”的机器，而是把市场里的钱分成三层来看： 宏观有没有放水，二级市场有没有足够成交和深度，SOL、Base、ETH 主网、BSC 哪条链真正有承接。

这是一份解释材料，不构成投资建议。它帮助你理解信号面板如何观察市场状态，而不是告诉你买什么或卖什么。

为什么不是做一个万能预测器

对个人交易者和开发者来说，最划算的第一步不是做复杂黑箱模型，而是做一个能持续更新、能解释原因的信号面板。

问题一

市场有没有钱

如果宏观流动性在收紧，市场即使短期上涨，也可能只是局部行情，承载面会比较弱。

问题二

钱在哪一层

钱可能在现货、合约、链上新项目之间移动，也可能在 SOL、Base、ETH 主网、BSC 之间轮动。只看总成交量，很容易看不清真正的驱动力。

问题三

热度能不能持续

如果只有 OI 和资金费率很热，但深度和链上承接没有跟上，行情更像短线拥挤。

面板只抓三层信号

小白可以把它理解成三张体检表：一张看大环境，一张看交易市场，一张看链上承接。

Macro

宏观流动性

主要看 TGA、稳定币供给、交易所余额和净流。它回答的是“大环境有没有变松”。

Secondary

二级市场流动性

主要看 CEX 深度、现货成交量、合约 OI、资金费率、合约/现货量比。它回答的是“山寨普涨有没有交易承接”。

68/100 示例
Primary

一级链上流动性

主要看 SOL、Base、ETH 主网、BSC 各自的活跃地址、链上交易量、DEX TVL、DEX 成交量、桥流入和链上稳定币供给。它回答的是“哪条链真正能承接新项目”。

71/100 示例

一级链上不能只看一个总分

总分只能告诉你“链上整体热不热”。真正有用的是拆开看：SOL、Base、ETH 主网、BSC 谁在升温，谁在退潮。

SOL

看 DEX 量效、稳定币供给、活跃地址、新池子和 SOL 相关二级交易是否一起升温。

Base

Base 没有原生交易代币，所以更依赖链上 DEX、桥净流入、稳定币和生态 token basket。

ETH 主网

看大体量资金是否仍停留在主网，还是开始向更活跃的链迁移。

BSC

看 BNB 相关二级活跃、BSC DEX 成交和链上稳定币是否同步改善。

最后合成四种市场状态

面板最终不应该只给一堆数字，而要把数字翻译成状态标签，并说明为什么变成这个状态。

流动性枯竭

三层都偏弱。资金、换手和链上承接都不足，不适合把局部反弹理解成全面行情。

流动性恢复

部分指标开始修复，但还没有形成共振。更像市场从冷到暖的过渡期。

流动性扩张

宏观、二级和链上信号同步改善。最接近“风险偏好打开”的阶段。

流动性过热

热度很高，但可能已经拥挤。尤其要看资金费率、OI 和深度是否开始给出风险信号。

最关键的是这张二维表

“山寨普涨”和“一级项目能打高估值”不是同一件事。前者更看二级市场，后者更看链上承接。

还要看游资怎么迁移

面板不应该只说“热”或“不热”，还要提示钱可能从哪里流向哪里。这里的“迁移”是根据信号推断的倾向，不是确定追踪每一笔钱。

二级市场 一级链上

当合约拥挤降温，但桥净流入、链上稳定币、DEX 量效和新池活动上升，说明交易资金可能向链上扩散。

一级链上 二级市场

当链上承接走弱，但 CEX 生态 token 的成交、OI 和资金费率升温，说明资金可能回到二级炒作。

ETH 主网 SOL / Base / BSC

当一条链桥流出和活跃度下降，另一条链桥流入、DEX 量效和稳定币供给上升，就可能发生链间轮动。

开发时怎么分阶段做

不要一开始就接所有数据源。先跑通最小闭环，再逐步扩展。

先统一名字

把交易所、链、资产、交易对的命名统一。比如 BTCUSDT、BTC-USDT、BTC-USDT-SWAP 都要能映射到同一个资产关系。

接最关键数据

宏观接 TGA，二级接 Binance 和 OKX，链上先固定 SOL、Base、ETH 主网、BSC，接 DeFiLlama、Coin Metrics 和桥/稳定币数据。第一版不追求全，只追求稳定和可解释。

先算四条链，再算总分

先分别计算 SOL、Base、ETH 主网、BSC 的链级一级分数，再汇总成一级总分，最后和宏观、二级一起判断市场状态。

输出状态和原因

面板不只显示“扩张”或“过热”，还要说明是什么指标推高或压低了分数。

再做告警和回测

等状态逻辑稳定后，再加状态切换告警和历史事件研究，用历史数据调整阈值。

最容易踩的坑

把合约热度误判成全面行情

OI 快速上升、资金费率很高，不一定是健康扩张。还要看现货成交、订单簿深度和链上承接。

把所有指标混成一个总分

总分可以有，但必须保留宏观、二级、一级链上三个子分数，否则看不出钱到底流向哪里。

第一版就做复杂模型

在线系统先用可解释规则。复杂统计检验适合放到离线研究里，用来校准权重和阈值。

过早接交易权限

这套系统第一版只需要只读数据。不要接私钥，不要接交易权限，不要把前端变成下单工具。
