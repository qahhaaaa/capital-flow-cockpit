① 核心心智模型

一句话:宏观决定水温 → 四层资金流决定往哪流 → 合成每个标的该不该上、上多少。

五层各自独立取真实免费数据、算出一个 方向 × 强度 × 置信度 的信号,再由引擎加权合成。任何一层取不到数据 → 标 missing,绝不用 0 冒充,也不污染其它层。

层回答什么数据源(免费)

L2 链间钱在哪条链上聚集/迁移DeFiLlama 稳定币份额
L3 发射台打新资金在哪个台子升温DeFiLlama 协议收入
L4 DEX↔CEX钱偏现货还是合约、拥不拥挤OKX / Hyperliquid 衍生品
L5 主题哪个板块/叙事在轮动DeFiLlama 板块 TVL

另有两条旁路(不进引擎,仅参考):稳定币总量潮汐(钱进出整个 crypto 池子)、App 收入热度(链上活动热度)。

② 颜色就是好坏 —— 看色不看字也能懂

绿 = 利好风险资产 / 钱在流入 / 升温 / 偏现货(承接型健康钱) / 放水 / 多头共振
红 = 利空 / 钱在流出 / 降温 / 合约过度拥挤 / 收水 / 空头共振
黄 = 中性 / 持平 / 分歧 / 只够"试探" / 数据部分可用(partial) / 合约偏热
灰 = 未知 / 数据缺失(missing)—— 看到灰色就知道"这块现在没数,别当 0 用"

④ 仓位档是怎么算出来的(最关键)

每个标的的 conviction(0–100) = 各层顺风分减逆风分,再乘宏观闸门系数:

单层得分 = 权重 × (该层强度/100) × 置信度系数
权重:链 0.3 · 发射台 0.3 · 主题 0.2 · DEX↔CEX 0.2　|　置信度:高=1 中=0.6 低=0.3
conviction = clamp( (顺风Σ − 逆风Σ) × 100 × 宏观系数 , 0, 100)　宏观系数:放水 1.1 · 中性 1 · 未知 0.85 · 收水 0.5
档位:≥75 标准　≥50 小仓　≥25 试探　否则 空仓

再叠加三条硬规则(顺序很重要):

  • 发射台硬下限:某台升温 且非收水 → 该台标的至少 试探(哪怕分数很低)。
  • 风险降档:每个风险标记(如合约拥挤、小盘出场流动性差)往下降一档。
  • 收水封顶:宏观收水时,任何标的最高只能 试探(最后执行,一票压制)。

⑤ 每层怎么读(数值好坏对照)

层 / 指标怎么算好(绿)差 / 警惕(红)
L1 净流动性美联储资产负债表 − TGA − RRP环比上行 = 放水下行 = 收水(压制所有仓位)
L2 份额变化(pp)该链稳定币占全局份额、按真实 4h 锚定的变化>0 = 资金净迁入该链<0 = 资金流出;"拐点"=慢漂移预警
L3 24h 收入 / 动量发射台协议真金白银收入 + 环比动量收入高 + 动量>0 = 打新主战场升温动量<0 = 降温
L4 funding / perp-spot合约资金费率 + 合约/现货成交比偏现货 = 承接型健康钱funding 高 + 比值高 = 挤在合约,过热防挤仓
L5 板块 7d TVL各板块 TVL 的 7 天相对强弱>0 = 叙事资金轮入<0 = 轮出

L4 说明:资金费率(funding)>0 表示做多的人在向做空的人付费 → 多头拥挤、合约过热,是提示防挤仓的黄灯,不是看多信号。真正健康的是钱回到现货去承接。

⑥ 这些数据可信吗 —— 看「可信度栏」

结论卡正下方那条就是可信度栏,三样东西一眼确认:

  • 更新时间:「X 分钟前」。每小时自动采一次;超过 90 分钟变黄、超 3 小时变红提示可能滞后。
  • 五个质量灯:宏观/链间/发射台/DEX↔CEX/主题,绿=ok 黄=partial 红=missing。
  • 点开看源:每个数据源(FRED/DeFiLlama/OKX…)的实时成功/失败状态都列出来,失败如实标注,不藏。

数据诚实是这个项目的铁律:取不到就标 missing/partial,绝不用 0 或样例数字冒充。所以你看到灰色/缺失时,那是真话,不是 bug。

⑦ 必须知道的边界

  • 只辅助、不下单:所有 conviction 与档位都是观察建议,交易可用等级上限 watch_only,永不接私钥、永不自动交易。
  • 阈值是相对的:强度基于滚动分位 / 变化率,不绑死绝对数字(绝对阈值易被刷、易过时)。刚起步历史点少时,强度/方向会显示 partial,需累积(方向≥4h、ok≥24h、潮汐 7d 指标≥7 天)。
  • 代理指标要打折:链间用"稳定币份额"代理资金迁移(真实桥净流是付费的);主题层的"热门搜索"只是注意力代理、可被操纵,不进引擎。
  • 顶部宏观三曲线(大科技 capex / ARR / 存储价)是手工维护、非实时的景气背景,已折叠在页面最底部,不进五层信号。
